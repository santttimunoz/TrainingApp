Archiving-related Configuration Parameters

Guidewire provides a set of configuration parameters that relate to the archive process. You use these parameters to enable and manage various aspects of the archive process. You set these configuration parameters in file config.xml.
 
Parameter
Type
Description
ArchiveEnabled
Boolean
Required. Whether archiving is enabled (set to true) or disabled (set to false). Default is false.
This parameter controls the creation of indexes on the ArchivePartition column. If set to true, PolicyCenter creates a non-unique index on that column for Extractable entities.
In PolicyCenter, this parameter controls whether or not the domain graph is started.
WARNING If you set ArchiveEnabled to true, the server refuses to start if you subsequently set it to false.
See “ArchiveEnabled”.
ArchiveDaysRetrievedBeforeArchive
Integer
Minimum number of days that must pass after retrieving a term before it can be archived again.
The default in the base configuration is 90.
See “ArchiveDaysRetrievedBeforeArchive”.
ArchiveDefaultRecheckDays
Integer
The minimum number of days that must pass before a policy term can be reconsidered for archiving after a previous check did not identify it as an archiving candidate.
The default in the base configuration is 30.
See “ArchiveDefaultRecheckDays”.
ArchivePolicyTermDays
Integer
The minimum number of days that must pass before a given policy term can be archived. The minimum number of days begins after the PeriodEnd for policies that have at least one bound period and after the close date for withdrawn or not taken policies.
The default in the base configuration is 366.
See “ArchivePolicyTermDays”.
ArchiveRecentJobCompletionDays
Integer
The minimum number of days that must pass after a job is closed before archiving the associated policy term. Jobs are associated with policy periods.
The default in the base configuration is 90.
See “ArchiveRecentJobCompletionDays”.
