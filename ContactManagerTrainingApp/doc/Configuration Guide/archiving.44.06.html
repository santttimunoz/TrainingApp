Batch Processes and Archiving

PolicyCenter has batch processes that archive policy terms and retrieve policy terms.
 
Name
Code
Description
Archive policy terms
ArchivePolicyTerm
Run this batch process to archive policy terms. This batch process calls the IPCArchivingPlugin which determines whether a policy can be archived.
For a policy period to be eligible for archiving, the server time must have reached the PolicyTerm.NextArchiveCheckDate date:
•
For more information on policy term eligibility, see “Selecting Policy Terms for Archive Eligibility”.
•
For information on the archive work queue, and those policy periods that the archive process archived, excluded, or skipped, see “Archive Info” in the System Administration Guide.
After running this work queue, Guidewire recommends that you update database statistics. This work queue makes large changes to the tables. Updating database statistics enables the optimizer to pick better queries based on more current data. For instructions to gather database statistics, see “Configuring Database Statistics” in the System Administration Guide.
For more information, see “Archiving Policy Terms” in the Application Guide and “Archiving-related Configuration Parameters”.
Retrieve policy terms
RestorePolicyTerm
Run this batch process to retrieve archived policy terms that are marked for retrieval. You can request retrieval of an archived policy term. Retrieving a policy term generates an activity for the user who requested the retrieval. For more information, see “Retrieving Archived Policies” in the Application Guide.
Note: PolicyCenter does not use the Archiving Item writer work queue.
