Generic Schedule Overview

In insurance terminology, schedules are lists of items that define, extend, restrict, or exclude coverages or coverage terms. The user interface for a generic schedule appears when you add or select the coverage, exclusion, or condition that requires a schedule. The user interface for a schedule is a table containing one row per scheduled item and as many columns as needed to capture the information pertaining to the scheduled item. Add and Remove buttons manage the schedule enabling it to contain any number of scheduled items, and to change over the life of the policy.
Scheduled coverage summaries also appear in the Policy Review screen, including the Differences card view. The data model for generic schedules enables you to configure schedules using a common set of Gosu classes, PCF pages, and entity interfaces.
In the base configuration, the General Liability line uses generic schedules. Schedules appear in other policy lines but are not implemented using generic schedules.
Schedules that have historically been a part of base product lines other than General Liability do not use the generic schedule data model.
